# The Clock From Hell™

## Component Chosen

🕐 Clock / Time Display

## Description

A clock is supposed to instantly show the current time.

My version intentionally makes the process frustrating by forcing the user through several unnecessary confirmation screens before revealing the time.

The clock still functions correctly and displays the real current time, but it introduces several bad user experience patterns.

## Intentionally Bad Features

### 1. Multiple Confirmation Screens

Users must pass through several steps before seeing the time.

### 2. Randomly Moving Clock

The clock changes position every second, making it harder to read.

### 3. Random Font Sizes

The clock constantly changes size for no useful reason.

### 4. Flashing Background Colors

The page background changes color every second.

### 5. Guilt Messages

The interface displays passive-aggressive messages such as:

* "You could be studying."
* "Another second gone forever."
* "Productivity not detected."

### 6. Meaningless Questions

Users are asked why they need the time even though the answer has no effect.

## Why This Is Bad UX

The design intentionally violates common usability principles:

* Efficiency
* Consistency
* Simplicity
* Predictability
* Accessibility

The result is a functional but frustrating clock experience.
